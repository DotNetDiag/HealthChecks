DotNetDiag HealthChecks continues the work of the original project and depends on contributors for new providers, fixes, tests, and documentation improvements.

## Contribution checklist

1. Read and follow [Don't push your pull requests](https://www.igvita.com/2011/12/19/dont-push-your-pull-requests/).
2. Follow the repository code guidelines and conventions.
3. Treat tests and documentation as part of the feature, not as optional follow-up work.
4. Review issues tagged [Ups for grabs](https://github.com/DotNetDiag/HealthChecks/labels/Ups%20for%20grabs), [help wanted](https://github.com/DotNetDiag/HealthChecks/labels/help%20wanted), and [question](https://github.com/DotNetDiag/HealthChecks/labels/question).

## Project links

- [Repository](https://github.com/DotNetDiag/HealthChecks)
- [Open issues](https://github.com/DotNetDiag/HealthChecks/issues)
- [Contributors graph](https://github.com/DotNetDiag/HealthChecks/graphs/contributors)

If you are contributing a new health check package, update the package documentation when it exists, update the relevant manual chapter, and keep the generated package reference section buildable.
